1. What is spec file in MEAN stack

In the MEAN stack, a spec file in Angular is used for testing the functionality of components, services, and other parts of the application. It has the .spec.ts extension and is automatically created when generating components or services using the Angular CLI. These files contain test cases written with testing frameworks like Jasmine and are run using tools like Karma. Spec files help ensure the code works as expected by checking for errors early in development. For example, when you create a component, Angular generates a spec file to test if the component loads correctly or if its data is handled properly. You can run all these tests using the ng test command.

R&D Questions

1. Why git is not showing git commit command while adding a file and it is showing command for unstage ?

Git doesn't automatically show the git commit command after you add a file because it expects you to manually run it. After using git add to stage a file, Git is waiting for you to either commit the changes using git commit or unstage them if you decide not to commit. If you're seeing unstage commands, it just means Git is letting you know that the file is staged and you can either commit it or unstage it.

2. What is the difference before first push and after first push ?

Before the first push, your code only exists on your computer in your local Git repository, and it's not yet uploaded to a remote service like GitHub. After the first push, your code is uploaded to the remote repository, making it visible and accessible online. This allows others to view or collaborate on your project, and your local changes are synced with the remote repository.

3. How the title tag is fetched to head and h1 is fetched to body when rendering a HTML file which has no structure ?

When rendering an HTML file with no structure, the browser automatically places the title tag into the head and the h1 tag into the body based on HTML parsing rules and defaults.

4. What is the difference between href and src and why it is used in img and anchor tag ?

The href and src attributes are used to link different resources in HTML but in different ways. The href attribute is used in the (anchor) tag to link to another webpage or file, so when you click the link, it takes you to that page. The src attribute is used in the tag to load and display an image on the webpage. In simple terms, href is for links you can click, and src is for loading and showing things like images.

5. Why there is three dots in if...else and do...while ?

The three dots (...) in if...else and do...while in documentation or examples are placeholders used to indicate omitted code or logic. They imply that additional statements or code blocks should be added in those places.

6. What is the difference between spread and rest operator ?

The spread operator (...) expands an array or object, while the rest operator (...) collects multiple elements into an array or parameters and used as function parameter.
